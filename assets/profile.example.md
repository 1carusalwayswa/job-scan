# 求职画像（示例模板）

> 复制为 `assets/profile.md` 并填入你自己的信息。`profile.md` 已被 .gitignore，不会进仓。
> 评分步骤会读这份画像，对照岗位 JD 打分。写得越具体，打分越准。

## 基本定位
- 目标级别：例如 mid-level（避开要求 5+ 年的资深岗）
- 所在地 / 可通勤地点：例如 Lund（用于 HTML 的 📍 就地标记，可选）
- 工作语言能力：例如「英语流利；不具备瑞典语工作能力」——这条直接影响语言门槛

## 核心技能
- 列出你最强的 3–6 项技术栈与领域关键词（用于技能匹配打分）

## 赛道偏好（与 search_config.json 的 lanes 对应）
- 对每条赛道写一句「为什么适合 / 权重高低」，例如：
  - Backend（主力）：有 X 年经验，已验证能转化
  - Data & ML（重仓）：差异化方向
  - …

## 加分 / 减分信号
- 加分：例如 远程友好、特定行业
- 减分：例如 纯硬件、要求母语级当地语言
